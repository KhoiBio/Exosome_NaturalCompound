# ExoRAG — Exosome · Cosmetic · Natural Compound Intelligence
### A fully local RAG pipeline using PubMed + ChromaDB + Claude AI

---

## What This Is

ExoRAG mines PubMed literature on exosomes, cosmetic applications, and natural compounds —
then lets you ask product development questions in plain English. Every answer is grounded
in real papers with clickable PubMed citations.

**Architecture:**
```
PubMed (free API) → abstracts.json → ChromaDB (local) ← sentence-transformers (local)
                                           ↓
                              Query → embed → similarity search
                                           ↓
                              Context + Prompt → Claude API → Answer
```

**Cost:**
| Component       | Cost         |
|----------------|--------------|
| PubMed fetch    | Free         |
| ChromaDB        | Free (local) |
| Embeddings      | Free (local) |
| Claude LLM      | ~$0.01/query |

## Prerequisites

- Python 3.10 or higher
- **An LLM — you must set up one of these before the app will work:**
  - **Option A — Claude API** *(recommended, best quality)*: Create a free account at https://console.anthropic.com → API Keys → Create Key. Add it to `.env` as `ANTHROPIC_API_KEY=sk-ant-...`
  - **Option B — Ollama** *(100% free, no account needed)*: Download from https://ollama.com, then open a terminal and run `ollama pull llama3.2`, then `ollama serve`. The app auto-detects it.
- ~1GB disk space (ChromaDB + embedding model)
- Internet connection (for PubMed fetch and Claude API only — everything else is local)

## Step-by-Step Setup

### Step 0 — Clone / download and set up Python environment

```bash
# Navigate to the project folder
cd exorag

# Create a virtual environment (keeps dependencies isolated)
python -m venv venv

# Activate it
# Mac / Linux:
source venv/bin/activate

# Windows:
# might need Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser prior to activate
venv\Scripts\activate

# Install all dependencies
pip install -r requirements.txt
```

**Note for Mac Apple Silicon (M1/M2/M3):** If you get errors with chromadb, try:
```bash
pip install chromadb --no-binary chromadb
```

---

### Step 1 — Configure your API key

```bash
# Copy the template
cp .env.example .env

# Open .env in any text editor and fill in:
#   ANTHROPIC_API_KEY=sk-ant-...   ← your Claude API key
#   ENTREZ_EMAIL=your@email.com    ← your real email (NCBI requires this)
```

**Where to get your Claude API key:**
1. Go to https://console.anthropic.com/
2. Click "API Keys" in the left sidebar
3. Click "Create Key"
4. Copy the key — it starts with `sk-ant-`
5. Paste it into `.env`

---

### Step 2 — Fetch PubMed literature

```bash
python scripts/01_fetch_pubmed.py
```

**What it does:**
- Searches 13 targeted PubMed queries covering:
  - Exosome + skin / cosmetic / anti-aging
  - Exosome + collagen, elastin, hyaluronic acid
  - Exosome + hair growth
  - Natural compounds (curcumin, quercetin, resveratrol, ginger, EGCG, etc.) + exosome
  - Plant-derived exosome-like nanoparticles
  - Exosome drug loading / formulation
  - Topical / transdermal exosome delivery
  - Cosmetic actives (retinol, vitamin C, niacinamide) + exosome
  - Melanin / pigmentation + exosome

**Output:** `data/abstracts.json` (~400–700 papers)

**Runtime:** 5–10 minutes (NCBI rate limits the API to 3 requests/second)

**No API key needed for this step.**

---

### Step 3 — Build the local vector index

```bash
python scripts/02_build_index.py
```

**What it does:**
- Loads `data/abstracts.json`
- Chunks each abstract into ~380-word segments with 50-word overlap
- Downloads `all-MiniLM-L6-v2` embedding model (~80MB, first run only)
- Embeds all chunks locally on your CPU
- Stores everything in `chroma_db/` (a local folder on your machine)

**Output:** `chroma_db/` folder with persistent vector store

**Runtime:**
- First run: 15–25 minutes on CPU (model download + embedding ~1000 chunks)
- Subsequent runs: ~5 minutes (no re-download)

**Cost: Free. Runs entirely on your machine.**

**Tip:** Run this overnight or while doing something else — it's a one-time setup.

---

### Step 4 — Test with the CLI (optional but recommended)

```bash
python scripts/03_query_cli.py
```

This opens an interactive prompt. Type a number to run a demo query, or type your own question.

**Good test queries:**
- `1` → "Which natural compounds in exosomes show the strongest anti-aging skin evidence?"
- `2` → "What plant-derived exosome-like nanoparticles work best for cosmetic delivery?"
- `3` → "How are lipophilic compounds like curcumin loaded into exosomes for skincare?"

Each query costs ~$0.001–0.003 with claude-3-5-haiku.

---

### Step 5 — Launch the Streamlit demo app

```bash
streamlit run scripts/04_app.py
```

Opens your browser at: **http://localhost:8501**

This is the full UI — use this for demos and interviews.

**Features:**
- 8 pre-built demo queries (click to run instantly)
- Streaming Claude answers with real-time typing effect
- Source panel with all retrieved papers + PubMed links
- Relevance scores for each source
- Session query history

---

### Step 6 — Run corpus analytics (optional)

```bash
python scripts/05_explorer.py
```

Outputs:
- Which compounds appear most in the literature
- Publication trend by year (is this field growing?)
- Top therapeutic indications
- Delivery route frequency
- Top journals in this space

Saves to `outputs/report.txt` — useful for interview slides.

## Changing the Claude Model

Edit `.env`:

```bash
# Fastest, cheapest (~$0.001/query) — good for demo
CLAUDE_MODEL=claude-3-5-haiku-20241022

# Smarter, slightly more expensive (~$0.01/query) — better answers
CLAUDE_MODEL=claude-3-5-sonnet-20241022
```

## Troubleshooting

**"chroma_db not found" error**
→ Run `python scripts/02_build_index.py` first

**"abstracts.json not found" error**
→ Run `python scripts/01_fetch_pubmed.py` first

**"ANTHROPIC_API_KEY not set" error**
→ Make sure `.env` exists and has your key (copy from `.env.example`)

**Slow embedding on first run**
→ Normal — the model downloads ~80MB and embeds ~1000 chunks. Runs once, then instant.

**ChromaDB install error on Mac M1/M2**
→ `pip install chromadb --no-binary chromadb`

**Port 8501 already in use**
→ `streamlit run scripts/04_app.py --server.port 8502`

**sentence-transformers install slow**
→ It installs PyTorch as a dependency — this is normal, can take a few minutes

## Interview Talking Points

1. **"This is fully local"** — ChromaDB and embeddings run on your laptop, no data leaves your machine except for the Claude API call

2. **"Built in 2 days"** — PubMed fetch → local embeddings → ChromaDB → Streamlit, all with existing tools I already know (LangChain architecture, ChromaDB, sentence-transformers)

3. **"Scalable to your internal data"** — same pipeline can index your internal formulation documents, lab notebooks, patents alongside PubMed — scientists can ask questions across all of it

4. **"Every answer is cited"** — no hallucinations about specific compounds, because the LLM is synthesizing real retrieved text with visible PMID sources

5. **"Re-indexes automatically"** — schedule `01_fetch_pubmed.py` monthly to stay current with new literature

6. **Prior validation** — built the same architecture for GBM radioresistance literature at USC; this is a proven pattern applied to a new domain
